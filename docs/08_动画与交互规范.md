# TeamSync 动画与交互规范

## 📋 文档信息
- **项目名称**: TeamSync 团队协作管理系统
- **版本**: v1.0
- **平台**: Flutter Web (桌面端)
- **日期**: 2026-02-10

---



**1. 设计核心原则**
系统的动画设计旨在引导用户注意力、提供即时反馈、增强界面的流畅感并减少等待焦虑。规范强调每个动画都必须有明确的目的，绝不为了动画而动画。核心原则包括：
- **快速响应**：大多数动画应在 200-300ms 内完成，避免拖沓。
- **物理自然**：通过缓动函数模拟现实世界的运动逻辑。
- **体验一致**：同类操作必须使用相同的动画时长和曲线。
- **无障碍支持**：尊重用户的减弱动态效果设置。

**2. 时长与缓动标准**
规范将动画时长细分为多个等级：
- **快速(150ms)**：用于按钮悬停等微小状态变化。
- **标准(200-300ms)**：适用于大多数 UI 过渡和页面切换。
- **进入(300-400ms)** 与 **退出(200ms)**：进入稍慢以引导视觉，退出较快以保持清爽。
- **缓动曲线**：进入或展开使用 `Ease Out`（减速），离开或收起使用 `Ease In`（加速），状态切换使用 `Ease In Out`，而点击反馈则推荐使用带弹性的 `Spring` 效果。

**3. 组件交互细节**
- **按钮**：悬停时背景加深、轻微上浮 1px 并伴随阴影和图标放大；点击时整体轻微缩小（97%）以产生物理按压感。
- **输入框**：聚焦时标签文字变色加粗、边框加粗并添加投影；错误提示通过弹簧曲线和淡入效果滑出。
- **卡片**：悬停时上移 2px 并加深投影，强化层级感。

**4. 页面与列表过渡**
- **页面切换**：采用淡入淡出（Fade）或轻微的水平位移（Slide）组合，使切换过程平滑不生硬。
- **列表加载**：列表项进入时采用交错延迟（每项延迟 50ms），配合从下往上的位移和淡入，形成韵律感。

**5. 反馈与微交互**
- **加载状态**：提供按钮内的环形进度条和骨架屏（Skeleton）规范，骨架屏使用 1.5s 循环的流光渐变效果。
- **提示反馈**：Toast 提示从顶部弹簧式滑入；复选框勾选时有缩放和颜色填充的联动；通知徽章则使用弹跳效果吸引注意。
- **手势反馈**：统一使用主色 20% 透明度的涟漪效果（Ripple）。

**6. 性能优化建议**
为了保证 Flutter Web 端的流畅度，规范建议：
- 尽可能使用 `const` 构造函数。
- 对复杂动画组件使用 `RepaintBoundary` 进行重绘隔离。
- 优先使用 `AnimatedBuilder` 而不是 `setState` 来驱动动画，以避免不必要的子组件重建。

通过这些细致的规范，TeamSync 能够确保在不同模块间提供一致、高效且富有生命力的用户体验。
