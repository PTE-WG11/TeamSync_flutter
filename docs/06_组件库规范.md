# TeamSync 组件库规范

## 📋 文档信息
- **项目名称**: TeamSync 团队协作管理系统
- **版本**: v1.0
- **平台**: Flutter Web (桌面端)
- **日期**: 2026-02-10

---

## 1. 组件概述

组件库采用 **Atomic Design** 方法论，分为：
- **Atoms (原子)**: 按钮、输入框、图标、文字等基础元素
- **Molecules (分子)**: 搜索框、表单字段、状态徽章等组合元素
- **Organisms (有机体)**: 侧边栏、任务卡片、统计卡片等复杂组件
- **Templates (模板)**: 登录页模板、仪表盘模板等页面框架

---

## 2. Atoms (原子组件)

### 2.1 Button (按钮)

#### 按钮类型

| 类型 | 样式 | 用途 |
|------|------|------|
| **Primary** | 主色填充背景 | 主要操作 |
| **Secondary** | 灰色边框 | 次要操作 |
| **Outline** | 主色边框 | 辅助操作 |
| **Ghost** | 透明背景 | 低优先级操作 |
| **Danger** | 红色填充 | 删除等危险操作 |

#### 按钮尺寸

| 尺寸 | 高度 | 水平内边距 | 字体 |
|------|------|-----------|------|
| **Large** | 48px | 24px | 16px Medium |
| **Medium** | 40px | 20px | 14px Medium |
| **Small** | 32px | 16px | 13px Medium |

#### 按钮状态

| 状态 | 样式变化 |
|------|---------|
| **Default** | 正常显示 |
| **Hover** | 背景加深 10%，轻微上浮 |
| **Pressed** | 背景加深 20%，scale 0.98 |
| **Loading** | 显示加载动画，禁用点击 |
| **Disabled** | 透明度 50%，cursor: not-allowed |

- **按钮 (Button)**：
    - **类型齐全**：提供主操作（Primary）、次要操作（Secondary）、描边（Outline）、虚幻（Ghost）及危险（Danger）五种类型。
    - **尺寸严谨**：定义了大、中、小三个尺寸，分别对应 48px、40px 和 32px 的高度。
    - **交互反馈**：内置了悬停加深、点击缩放（0.98）、加载中动画以及禁用状态的视觉处理。
---

### 2.2 Input (输入框)

#### 输入框类型

| 类型 | 用途 |
|------|------|
| **Text** | 单行文本输入 |
| **Password** | 密码输入，支持显示/隐藏切换 |
| **Email** | 邮箱输入，自动验证格式 |
| **Number** | 数字输入 |
| **TextArea** | 多行文本输入 |
| **Search** | 搜索输入，带清除按钮 |

#### 输入框状态

| 状态 | 样式 |
|------|------|
| **Default** | 灰色边框 #E2E8F0 |
| **Hover** | 边框加深 #CBD5E1 |
| **Focus** | 主色边框 #0D9488，宽度 2px |
| **Error** | 错误色边框 #F43F5E |
| **Disabled** | 灰色背景，禁用交互 |
| **Filled** | 绿色对勾图标提示 |

   
- **输入框 (Input)**：
    - **功能丰富**：涵盖文本、密码（支持显隐切换）、邮箱、搜索（带清除按钮）等多种模式。
    - **状态分分明**：通过边框颜色和投影区分默认、聚焦（2px 主色边框 + 阴影）、错误（红色）及禁用状态。
- **复选框 (Checkbox)**：
    - 支持带标签的点击交互，选中时带有平滑的背景填充和对勾动画。

**3. 分子组件规范 (Molecules)**
- **徽章系统 (Badges)**：
    - **状态徽章**：用于展示任务状态（规划中、待处理、进行中、已完成），采用“浅色背景 + 深色文字 + 状态圆点”的组合。
    - **优先级徽章**：通过颜色直观区分紧急程度（红色-紧急、橙色-高、蓝色-中、灰色-低）。

**4. 有机体组件规范 (Organisms)**
- **统计卡片 (StatCard)**：
    - 专为仪表盘设计，包含数值、标签及左侧彩色装饰条。
    - **动态效果**：内置数字滚动动画（AnimatedCounter），增强数据展示的生动性。

**5. 开发实现标准**
- **统一配置**：所有组件严格遵循全局的 `AppColors`、`AppTypography` 和 `AppShadows` 配置。
- **Flutter 封装**：通过枚举（Enum）控制类型和尺寸，采用私有 Getter 集中处理样式逻辑，确保业务代码调用简单且表现一致。
- **交互细节**：广泛应用 `AnimatedContainer` 和 `InkWell`，保证 Web 端拥有丝滑的鼠标悬停与点击反馈。

通过这套规范，TeamSync 构建了一个从底层像素到高层逻辑都高度统一、易于扩展的组件化开发环境。
