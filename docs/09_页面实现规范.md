# TeamSync 页面实现规范

## 📋 文档信息
- **项目名称**: TeamSync 团队协作管理系统
- **版本**: v1.0
- **平台**: Flutter Web (桌面端)
- **日期**: 2026-02-10

---

**1. 核心页面体系**
规范明确了系统当前阶段的三个关键页面及其职责：
- **登录页**：用户的访问入口。
- **注册页**：新用户账号创建及流程指引。
- **成员首页**：核心业务看板，用于展示个人任务与项目进度。

**2. 登录页设计与实现**
登录页采用经典的左右分割布局：
- **左侧品牌区**：展示 TeamSync 品牌标识、系统 Slogan 以及三大核心特性（项目管理、权限控制、进度追踪），通过深色背景和直观图标强化专业形象。
- **右侧表单区**：包含品牌微标、欢迎语、登录表单（用户名/密码）、“记住我”勾选框及“忘记密码”链接。
- **交互逻辑**：表单内置校验机制，点击登录后进入加载状态并模拟 API 调用，成功后跳转至首页。

**3. 注册页设计与实现**
注册页延续了左右分割的视觉一致性：
- **左侧指引区**：侧重于流程说明，通过清晰的步骤卡片告知用户注册后的操作路径（注册 -> 登录 -> 等待团队邀请 -> 开始协作）。
- **右侧表单区**：提供用户名、邮箱、密码及确认密码的输入项。
- **校验逻辑**：除了基础的非空校验，还特别强调了两次密码输入的一致性检查。

**4. 代码组织与技术细节**
- **结构化组件**：页面被拆分为私有的局部组件（如 `_LeftPanel` 和 `_RightPanel`），提高了代码的可读性和维护性。
- **样式统一**：严格调用全局配置中的 `AppColors` 和 `AppTypography`，确保视觉风格与 UI 规范完全一致。
- **常用组件应用**：统一使用预封装的 `AppButton`、`AppTextField` 和 `AppCheckbox`，并充分利用 `GoRouter` 进行页面间的高效导航。
- **响应式与约束**：表单区域使用 `ConstrainedBox` 限制最大宽度（400px），确保在不同分辨率的屏幕上都能保持良好的视觉中心。
